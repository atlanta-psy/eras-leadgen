# Автопостинг тредов в Threads через Publora

Ставит твои треды в отложку в Threads по времени. Работает через API Publora — я (или автозадача) делаю это сам, коннектор не нужен.

## Один раз — настройка (на твоей стороне)
1. На publora.com завести аккаунт, поднять план **Pro** ($2.99/мес — на бесплатном Threads нет).
2. Подключить в Publora свой аккаунт **Threads**.
3. Раздел **API** → сгенерировать ключ `sk_...`.
4. Создать файл `publora_key.txt` рядом со скриптом и вписать туда **одну строку — этот ключ**. (В чат ключ не вставляй.)
5. Узнать id своего Threads-канала:
   ```
   python3 publora_post.py connections
   ```
   В выводе найти строку вида `threads-XXXX` и вписать её в `publora.yaml` → `threads_platform`.

## Формат файла тредов
Обычный `.md`. Цепочки тредов разделяются строкой из `===`, части внутри цепочки — строкой `---`:

```
первый пост цепочки
---
второй пост этой же цепочки
===
первый пост другой цепочки
```

## Поставить в отложку
```
python3 publora_post.py post --file treds.md --dry-run   # проверка: покажет, что и когда уйдёт
python3 publora_post.py post --file treds.md             # реально поставить в отложку
```
Время и количество в день настраиваются в `publora.yaml` (`default_times`, `per_day`, часовой пояс).

## Важно знать
- **Связанные цепочки тредов (вложенные посты) у Publora временно отключены** из-за переподключения Threads на их стороне. Одиночные посты и отложка работают. На тесте сразу увидим, встают ли цепочки; если нет — публикуем как одиночные треды, пока Publora не восстановит.
- Лимиты Threads: до 50 постов/час, 250/сутки. Каждая часть цепочки = отдельный пост в счёт лимита.
- `publora_key.txt` — никогда не коммить в репозиторий (он в .gitignore).

## Автопилот
Когда тест пройдёт, сделаем ежедневную автозадачу: утром берётся файл тредов и сам уходит в отложку — без твоего участия.
