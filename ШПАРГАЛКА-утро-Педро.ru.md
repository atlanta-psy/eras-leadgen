# шпаргалка: утро Педро (для Елены)

короткая памятка, чтобы помочь Педро, если позвонит с вопросом. команды он запускает из папки агента.

## один раз — подготовка базы (не каждый день)
- `python leadgen.py collect --niche guest_house` — собрать дома и контакты
- `python enrich.py --niche guest_house` — добрать почту/телеграм/вк с сайтов, убрать дубли
- `python offer.py --niche guest_house` — написать персональные офферы
после этого база готова, по ней рассылаем дни/недели.

## каждое утро — 4 шага (10 минут)
1. **почта** — `python send_manager.py send-email --niche guest_house` (компьютер сам разошлёт письма дня). первый раз можно с `--dry-run` — покажет, кому уйдёт, без отправки.
2. **телеграм/вк руками** — `python send_manager.py today --niche guest_house` → откроется файл `today_...csv` с тандой → копирует текст, отправляет со *специального* аккаунта.
3. **отметить ответивших** — `python send_manager.py replied --niche guest_house --name "Название"`.
4. **посмотреть картину** — `python send_manager.py dashboard --niche guest_house` → откроется excel.

фоллоу-апы (второе и третье касание) агент подкидывает сам в следующие дни.

## почту можно поставить на автопилот
файл `enviar-correo-diario.bat` + «Планировщик заданий» Windows = почта уходит сама каждое утро. (см. шаги в испанской шпаргалке Педро.) телеграм/вк всё равно руками.

## если что-то не так
- **почта не ушла** → проверить `configs/email_config.yaml` (пароль/сервер), либо сегодня лимит прогрева маленький, либо база на сегодня уже обработана.
- **today пусто** → телеграм/вк-лиды на сегодня кончились или исчерпан лимит — это норма.
- **хочет больше в день** → поднять цифры в `configs/send_config.yaml`, но только после первой недели прогрева.
- **кто ответил?** → ответы приходят в обычную почту/телеграм Педро, агент их не видит; он смотрит сам и отмечает `replied`.

## золотое правило
рабочие аккаунты в холодную рассылку не пускать. объём наращивать постепенно. тогда не забанят.
